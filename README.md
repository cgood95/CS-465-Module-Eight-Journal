# CS-465-Module-Eight-Journal

Architecture

Compare and contrast the types of frontend development you used in your full stack project, including Express HTML, JavaScript, and the single-page application (SPA).
Why did the backend use a NoSQL MongoDB database?

In my full stack project, I used multiple types of frontend development, including Express with HTML templates, JavaScript, and a single-page application (SPA). The Express HTML approach allowed me to render pages on the server side, which is useful for simpler page loads and initial structure. JavaScript enhanced interactivity on those pages by handling events and dynamic updates. In contrast, the SPA approach provided a smoother user experience by loading content dynamically without refreshing the entire page, making the application feel faster and more modern. Each method had its strengths, but the SPA was especially useful for creating a more responsive and seamless interface. The backend used a NoSQL MongoDB database because it is flexible and stores data in a JSON-like format, which works well with JavaScript-based applications. This made it easier to manage and scale data without needing a rigid schema.

Functionality

How is JSON different from Javascript and how does JSON tie together the frontend and backend development pieces?
Provide instances in the full stack process when you refactored code to improve functionality and efficiencies, and name the benefits that come from reusable user interface (UI) components.

JSON is different from JavaScript because it is a data format used for storing and transmitting data, while JavaScript is a programming language used to create logic and functionality. JSON plays a key role in connecting the frontend and backend because it is the format used to send and receive data through API calls. For example, when the frontend requests data from the server, the backend responds with JSON, which the frontend can then process and display. During the project, I refactored code multiple times, such as simplifying API calls and organizing functions into reusable modules. One major improvement was creating reusable UI components, which made the code more efficient and easier to maintain. The benefits of reusable components include reducing duplication, improving consistency across the application, and making future updates faster and easier.

Testing

Methods for request and retrieval necessitate various types of API testing of endpoints, in addition to the difficulties of testing with added layers of security. Explain your understanding of methods, endpoints, and security in a full stack application.

In a full stack application, methods, endpoints, and security are all important parts of API testing. Methods such as GET, POST, PUT, and DELETE are used to perform different operations on data, while endpoints are the specific routes where these requests are sent. Testing ensures that each endpoint works correctly and returns the expected results. When security is added, such as authentication for admin login, testing becomes more complex because you need to verify that only authorized users can access certain routes. This may involve testing tokens, login credentials, and protected routes. Understanding how these pieces work together helped me see the importance of secure and reliable communication between the frontend and backend.
